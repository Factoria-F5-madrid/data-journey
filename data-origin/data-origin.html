Estrategias de recolección e ingestión

La recolección e ingestión de datos representan la primera etapa en el ciclo de la información. Consiste en obtener datos desde diversas fuentes humanas, naturales o digitales y prepararlos para su procesamiento y análisis.

Fuentes Estructuradas: BBDD, APIs, ERP Fuentes no Estructuradas: Log, Sensores, Redes Sociales

Fuentes estructuradas y fuentes no estructuradas

Los datos estructurados tienen un formato predefinido (como tablas de bases de datos) que facilita su análisis. Por el contrario, los datos no estructurados no siguen un esquema fijo y vienen en formatos diversos como texto, imágenes y audio, siendo más complejos de analizar. Las fuentes comunes de datos estructurados son las bases de datos relacionales, mientras que las de datos no estructurados incluyen correos electrónicos, videos y redes sociales.

BBDD, APIs, ERP

- Las bases de datos relacionales usan un modelo de tablas estructurado (filas y columnas) para datos organizados y consistentes, mientras que las no relacionales (NoSQL) almacenan datos de manera más flexible usando modelos como documentos, grafos o pares clave-valor, ideal para datos no estructurados o grandes volúmenes. La principal diferencia está en su estructura de datos: fija en las relacionales y flexible en las no relacionales.

- ¿Qué es una fuente API? Una API es un intermediario que permite a dos programas de software comunicarse entre sí y define cómo solicitan y reciben datos o funcionalidades. Las API son esenciales para crear aplicaciones de software modernas que conectan y comparten información.

- Un ERP es una fuente de datos estructurados que integra y gestiona información de procesos clave de un negocio (como finanzas, RR. HH. y cadena de suministro) en una única base de datos. Estos datos son estructurados porque están organizados en un formato predefinido (por ejemplo, la información del cliente, los detalles de un pedido) que permite su fácil análisis y consulta a través de diferentes herramientas.

logs, sensores, redes sociales

- Los logs (registros) son una fuente de datos no estructurados porque registran actividades en un formato secuencial, generalmente de texto, sin un modelo predefinido. Estos archivos registran eventos como operaciones de usuarios, errores o procesos automáticos en sistemas, redes o aplicaciones, pero la información está en formato libre y de texto.

- La fuente de datos de sensores es la información bruta que recopilan los sensores al medir condiciones del entorno físico, químico o biológico. Estos datos se generan al convertir señales como temperatura, luz, presión o movimiento en datos electrónicos para su posterior almacenamiento y análisis.

- Las fuentes de datos de redes sociales pueden ser las APIs de las plataformas (Application Programming Interfaces) para recopilación programada, o datos extraídos manualmente de sitios como los informes de plataformas (ej. YouTube Culture & Trends), o herramientas de análisis como Hootsuite y Statista. También se pueden usar métodos como el web scraping.
