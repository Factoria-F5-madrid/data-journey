Limpieza & Transformación

La limpieza y transformación de datos consiste en preparar los datos brutos para su análisis. Incluye detectar y corregir errores, valores faltantes o duplicados, y convertir formatos o tipos de datos. Su objetivo es obtener un conjunto de datos coherente, preciso y listo para el modelado o la visualización.

Data Wrangling & Calidad de los datos

El Data Wrangling es el proceso de limpiar, transformar y organizar datos sin procesar para hacerlos útiles en análisis. La calidad de los datos se refiere a su precisión, completitud, consistencia y confiabilidad. Ambos conceptos están estrechamente relacionados: un buen Data Wrangling mejora la calidad de los datos, lo que es esencial para obtener resultados precisos en análisis, modelos predictivos y toma de decisiones

1600s

Leibniz y el cálculo lógico

Gottfried Wilhelm Leibniz soñó con crear una “máquina de razonamiento” que pudiera realizar deducciones lógicas, anticipando la noción de una computadora simbólica.

1800s

ETL/ELT y pipelines

Los pipelines ETL se limitan a extraer datos de fuentes, transformarlos en un formato utilizable y cargarlos en una base de datos o un almacén de datos. Por otro lado, la integración de datos puede implicar ETL, ELT, transmisión de datos en tiempo real, integraciones basadas en API y virtualización de datos.

1930s

Feature Engineering

La ingeniería de características (feature engineering) es el proceso de transformar datos brutos en características (variables) útiles que mejoran el rendimiento de los modelos de aprendizaje automático. Implica seleccionar, crear y modificar variables para hacer que los datos sean más adecuados para un modelo, utilizando conocimiento del dominio y técnicas matemáticas o estadísticas.
